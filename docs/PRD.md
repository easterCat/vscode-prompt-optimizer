## 1. 项目概述

### 1.1 项目背景
[Prompt Optimizer](https://github.com/linshenkx/prompt-optimizer) 是一个功能全面的AI提示词优化工具，支持Web、桌面应用、Chrome扩展等多种形式。本项目旨在提取其核心优化能力，开发一个轻量级VSCode插件，方便开发者在编码环境中快速优化提示词，而无需切换应用。

### 1.2 产品目标
开发一个VSCode插件，提供最精简、高效的AI提示词优化工作流：**配置模型 -> 输入原始提示词 -> 获得优化后的提示词**。

### 1.3 目标用户
- 使用VSCode进行AI应用开发、Prompt Engineering的开发者。
- 希望在编码环境中快速迭代和测试提示词的用户。
- 需要轻量级、无干扰优化工具的用户。

## 2. 功能范围

### 2.1 核心功能 (必须包含)
| 功能模块 | 描述 | 优先级 |
| :--- | :--- | :--- |
| **模型配置** | 允许用户配置AI模型（如OpenAI、Gemini等）的API密钥、Endpoint、模型名称等。 | **P0 (最高)** |
| **原始提示词输入** | 提供一个文本输入区域，让用户粘贴或编写原始的提示词。 | **P0** |
| **优化后提示词输出** | 展示优化后的提示词结果，支持一键复制。 | **P0** |
| **优化触发** | 一个明确的按钮（如“Optimize”），点击后调用配置好的模型对原始提示词进行优化。 | **P0** |

### 2.2 非功能需求 (必须满足)
| 需求类型 | 描述 |
| :--- | :--- |
| **安全性** | API密钥等敏感信息必须存储在VSCode的Secret Storage中，不得明文保存在任何配置文件中。 |
| **性能** | 优化请求应为异步非阻塞，避免卡顿VSCode界面。优化过程中应有加载状态提示。单次请求超时限制为 60 秒，超时后显示超时错误信息。 |
| **易用性** | 界面简洁，操作直观，可考虑与VSCode主题适配。 |

### 2.3 明确排除的功能 (不在此版本范围)
- 多轮迭代优化
- 提示词对比评估
- 提示词模板库 / 收藏夹
- 图片生成
- 多轮对话测试
- 上下文变量管理
- Function Calling
- MCP协议支持
- 多平台部署

## 3. 功能详细设计

### 3.1 模型配置 (Model Configuration)

**用户场景**：用户首次使用或需要更换优化模型时，配置API连接。

**界面与交互**：
- **入口**：在VSCode活动栏提供插件图标，点击后显示侧边栏视图。侧边栏顶部有一个”设置”图标⚙️或单独的”模型配置”区域。
- **配置项**：
    1.  **界面语言**：下拉选择 (`中文`, `English`)，切换后系统提示词和用户模板的默认值同步切换。
    2.  **模型提供商**：下拉选择 (`openai`, `gemini`, `deepseek`, `custom`)。切换提供商时自动填充对应的默认 Base URL 和模型名称。
    3.  **API Key**：密码输入框，输入时隐藏字符。
    4.  **Base URL (可选)**：文本框，默认为官方Endpoint，支持自定义（如代理或兼容API）。各提供商默认值：
        - `openai`: `https://api.openai.com/v1`
        - `gemini`: `https://generativelanguage.googleapis.com/v1beta`
        - `deepseek`: `https://api.deepseek.com/v1`
        - `custom`: 留空
    5.  **模型名称**：文本框，如 `gpt-4o`, `gemini-1.5-pro`。
    6.  **系统提示词 (可编辑)**：多行文本框，显示当前语言的默认优化系统提示词，用户可自定义。提供”重置”按钮恢复默认值。
    7.  **用户提示词模板 (可编辑)**：多行文本框，显示当前语言的默认用户提示词模板（含 `{prompt}` 占位符），用户可自定义。提供”重置”按钮恢复默认值。
    8.  **保存/测试**：按钮，保存配置并可选择发送测试请求验证有效性。测试通过发送 `”Hello”` 作为优化请求来验证 API 连通性。

**存储方案**：
- 使用 `vscode.SecretStorage` 存储 `API Key`（存储键：`easyPromptOptimizer.apiKey`）。
- 使用 `vscode.ExtensionContext.globalState` 存储其他非敏感配置（存储键：`easyPromptOptimizer.modelConfig`），包括：模型提供商、Base URL、模型名称、系统提示词、用户提示词模板、界面语言。
- **默认配置**：未配置时使用 OpenAI 提供商、`gpt-4o` 模型、中文界面。

### 3.2 原始提示词输入 (Original Prompt Input)

**用户场景**：用户编写好初始提示词，准备进行优化。

**界面与交互**：
- 侧边栏中提供一个**多行文本输入框**（VSCode的`Webview`中的`<textarea>`）。
- 支持从当前编辑器中选中的文本**一键填充**到输入框（在编辑器右键菜单增加“Send to Prompt Alchemy”）。
- 输入框支持基本的文本编辑（换行、复制粘贴）。

### 3.3 提示词优化与输出 (Optimization & Output)

**用户场景**：用户点击优化按钮，等待并获取结果。

**工作流程**：
1.  用户点击侧边栏中的 **“Optimize”** 按钮（蓝色主色调）。
2.  插件校验配置（API Key、模型名是否填写），若未配置则引导用户配置。
3.  插件发送异步请求到配置的API端点。
    - 请求使用 **OpenAI Chat Completions API** 格式（temperature: 0.7, max_tokens: 4096）。
    - **系统提示词**驱动模型改进用户提示词（默认中文版本定义 AI 为”用户提示词基础优化助手”，重点消除模糊表达、补充关键信息、提升清晰度）。用户可在配置区域自定义。
    - **用户提示词模板**将用户输入包装为 JSON 结构 `{ “originalPrompt”: “{prompt}” }`，并明确指示模型只优化文本本身，不执行提示词内容。用户可在配置区域自定义。
    - Gemini 提供商使用特殊的 API 路径 (`/models/{model}:generateContent`)，并**同时**通过 `x-goog-api-key` 请求头和 URL 查询参数 `?key=` 两种方式传递 API key 以确保兼容性。
4.  请求过程中，按钮变为“优化中...”并显示加载指示器，禁用输入和优化按钮。
5.  收到响应后，将优化后的提示词显示在**下方的只读文本区域**。
6.  输出区域提供 **“复制”** 按钮，一键复制到剪贴板。
7.  若请求失败（网络错误、API错误、认证失败），显示清晰的错误信息。

## 4. 技术约束与建议

| 项目 | 建议 |
| :--- | :--- |
| **开发语言** | TypeScript |
| **VSCode API** | 使用 `vscode` 扩展API，`Webview` 构建UI，`SecretStorage` 存储密钥。 |
| **HTTP客户端** | Node.js 原生 `https` / `http` 模块（零外部依赖）。 |
| **项目结构** | `src/extension.ts` (入口，注册侧边栏和命令)，`src/config/modelConfig.ts` (配置管理与持久化)，`src/optimizer/` (优化引擎、系统提示词、用户模板)，`src/webview/` (侧边栏 Provider 与内联 HTML/CSS/JS)。共 7 个源文件，约 1,451 行 TypeScript。 |
| **API兼容** | 优先支持 **OpenAI Chat Completions API** 格式，因为它已成为事实标准，可覆盖大多数代理和兼容模型（如DeepSeek、本地Ollama等）。 |
| **优化提示词** | 内置高质量的默认优化系统提示词和用户模板（中/英双语），同时已开放给用户在配置区域自定义编辑，并支持一键重置为默认值。 |

## 5. 用户界面草图 (文字描述)

**VSCode侧边栏 (Prompt Alchemy View) — 三个可折叠区域**

```
┌─────────────────────────────┐
│ ▼ 📝 原始提示词              │
│ ┌─────────────────────────┐ │
│ │ 写一首关于编程的诗       │ │
│ │                         │ │
│ └─────────────────────────┘ │
│             [ 优化 🚀 ]      │
├─────────────────────────────┤
│ ▼ ✨ 优化后的提示词    [复制] │
│ ┌─────────────────────────┐ │
│ │ 你是一位精通计算机科学   │ │
│ │ 与诗歌艺术的创作大师。   │ │
│ │ 请以”编程”为核心主题... │ │
│ └─────────────────────────┘ │
├─────────────────────────────┤
│ ▶ ⚙️ 模型配置               │
│   语言: [中文 ▼]             │
│   提供商: [OpenAI ▼]         │
│   API Key:  [••••••••]       │
│   Base URL: [https://api...] │
│   模型名:   [gpt-4o     ]    │
│   系统提示词: [... ]  [重置]  │
│   用户模板:   [... ]  [重置]  │
│   [保存]  [测试连接]         │
└─────────────────────────────┘
```

**交互细节**：
- 界面语言切换仅影响系统提示词和用户模板的默认值，**不改变 UI 控件标签**。UI 标签始终使用英文（如 "Optimize"、"Save"、"Test"、"Copy"）。下方草图中使用中文标注是为了提高 PRD 可读性，实际界面以英文为准。
- 三个区域均支持点击标题栏折叠/展开，带平滑动画。
- 切换提供商时自动填充默认 Base URL 和模型名称。
- 优化过程中按钮显示加载动画并禁用输入。
- 操作结果通过 Toast 通知提示（成功/失败）。
- 从编辑器右键菜单发送选中文本时，自动填充到输入区域。
